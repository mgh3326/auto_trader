`data/` now holds maintained Python loader modules for non-DB reference datasets.

## CONVENTIONS
- Keep runtime data-loading logic in Python modules (`get_*`, `prime_*` patterns).
- Keep package exports in `__init__.py` aligned with actual loader ownership.
- Prefer targeted loader updates over broad directory-wide changes.

## ANTI-PATTERNS
- Do not treat `__pycache__` contents as source of truth.
- Do not mix generated/runtime cache artifacts into maintained source workflows.
- Do not reintroduce static file-based Upbit market loaders for runtime logic.
- Do not reintroduce file-based KR/US stock universes; use DB sync services instead.

## NOTES
- KR/US/Upbit symbol sources are DB-backed and maintained through dedicated sync scripts/services, not under `data/`.
- Root `pyproject.toml` excludes data directories from Ruff checks, so local validation may need explicit targeted checks.
